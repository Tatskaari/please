Please 16.0.0 migration guide

This major release has focused mainly around distribution and installation.

Installation

In v15, Please used to be distributed with all its tools for the core languages: Go, Java, Python, and C++. In v16, Please is no longer distributed with all its tools. Instead, Please will download its tools on demand through build rules in the new internal //:_please, package.

Go modules and Go 1.16 support

Please v16 now support go 1.16 through use of the go_module() rules. go_get() will recompile certain packages from source instead of using the already compiled package. This doesn't work with 1.16 fingerprinting.

Going forward, go_get() will be deprecated in favour of go_module(). For the most part, they work essentially the same.
